# Task Service с поддержкой периодических задач

Данный сервис является расширенной версией [тестового задания](https://github.com/medods/test-task-for-junior-backend-developer).  
Добавлена возможность создавать задачи, которые будут автоматически повторяться по заданному расписанию.

## 📌 Реализованная функциональность

- Создание, редактирование, удаление и получение задач (CRUD).
- **Новое:** настройка периодичности задачи с помощью объекта `recurrence`.
- Автоматическая генерация экземпляров повторяющихся задач фоновым процессом (cron).

### Типы повторения

| Тип          | Параметры                                                     |
|--------------|---------------------------------------------------------------|
| `daily`      | `interval` – повторять каждые n дней (n ≥ 1)                  |
| `monthly`    | `day_of_month` – число месяца от 1 до 30                      |
| `specific`   | `dates` – массив конкретных дат в формате `YYYY-MM-DD`        |
| `parity`     | `parity` – `"even"` (чётные дни) или `"odd"` (нечётные дни)   |

## 🧠 Принятые архитектурные решения и предположения

### 1. Хранение шаблонов и экземпляров
- Одна таблица `tasks` используется и для обычных задач, и для шаблонов повторения, и для сгенерированных экземпляров.
- Добавлены поля:
  - `scheduled_at` – плановая дата выполнения (обязательное для всех задач).
  - `recurrence` (JSONB) – настройки повторения (только у шаблонов).
  - `template_id` – ссылка на шаблон (только у сгенерированных экземпляров).
- **Обоснование:** минимальные изменения существующей схемы, сохранение обратной совместимости.

### 2. Генерация задач
- Используется **фоновая cron-задача**, которая ежедневно в 00:01 UTC запускает метод `GenerateTasksForDate` для завтрашнего дня.
- **Почему не прегенерация на месяцы вперёд?**  
  - Экономия места в БД.
  - Возможность изменения шаблона без необходимости корректировать уже созданные экземпляры (как в Google Calendar).
- **Почему не динамическая генерация при запросе списка?**  
  - Усложнило бы фильтрацию, пагинацию и производительность.

### 3. Поведение при обновлении / удалении шаблона
- Изменение шаблона **не влияет** на уже созданные экземпляры.
- Удаление шаблона **не удаляет** его экземпляры (они остаются в истории).
- **Обоснование:** типичное поведение календарей – уже запланированные события не должны исчезать или меняться задним числом.

### 4. Обработка граничных случаев
- **30 февраля** – для monthly-задач с `day_of_month = 30` в феврале задача будет создана на последний день месяца (28 или 29).
- **Пропущенные дни из-за остановки сервиса** – cron генерирует задачи только на будущий день; пропущенные даты не заполняются.
- **Високосные годы** – корректно учитываются при расчёте дат.

### 5. Валидация recurrence
- Проверяется на уровне доменной модели (`Recurrence.Validate()`).
- При невалидных данных возвращается `400 Bad Request`.

## 📦 Запуск проекта

### Требования
- Docker и Docker Compose
- (опционально) Go 1.23+ для локальной разработки

### Быстрый старт

# Клонировать репозиторий
git clone <your-repo-url>
cd <project-dir>

# Запустить сервисы
docker compose up --build

Сервис будет доступен по адресу http://localhost:8080.
Swagger UI: http://localhost:8080/swagger/

### Миграции
При первом запуске PostgreSQL автоматически выполняет SQL-файлы из папки migrations/, создавая таблицу tasks с необходимыми колонками (scheduled_at, recurrence, template_id).

### 📡 API Примеры
Создание ежедневной задачи (каждые 2 дня)

POST /api/v1/tasks
Content-Type: application/json

{
  "title": "Обзвон пациентов",
  "description": "Плановый обзвон",
  "status": "new",
  "scheduled_at": "2025-05-01T09:00:00Z",
  "recurrence": {
    "type": "daily",
    "interval": 2
  }
}

### Создание задачи на конкретные даты

POST /api/v1/tasks
Content-Type: application/json

{
  "title": "Совещание",
  "scheduled_at": "2025-06-01T10:00:00Z",
  "recurrence": {
    "type": "specific",
    "dates": ["2025-06-01", "2025-06-15", "2025-07-01"]
  }
}
### Получение списка всех задач

GET /api/v1/tasks
Ответ содержит как шаблоны (с полем recurrence), так и обычные задачи, а также сгенерированные экземпляры (с template_id).

### 📝 Примечания по проверке cron
Для удобства тестирования можно временно изменить расписание cron в cmd/api/main.go:


_, err = c.AddFunc("*/1 * * * *", ...) // каждую минуту
После пересборки новые экземпляры будут появляться каждую минуту (для шаблонов, у которых scheduled_at ранее текущей даты).
